## 1. Регистрация на GitHub

1. Открой сайт: **https://github.com/**
2. Нажми **Sign up** (зелёная кнопка)
3. Введи email (действующий!)
4. Придумай пароль (минимум 8 символов, буквы + цифры)
5. Придумай username (например: `SashaCook`)
6. Нажми **Create account**
7. Пройди проверку (капча)
8. Проверь почту — открой письмо от GitHub и нажми **Verify email address**

---

## 2. Установка GitHub Desktop

1. Перейди на: **https://desktop.github.com/**
2. Нажми **Download for Windows**
4. Открой установщик и нажми **Install Now**
5. После установки открой приложение
6. Войди под своим GitHub аккаунтом (нажми **Sign in to GitHub.com**)

---

## 3. Создание репозитория

1. Открой GitHub Desktop
2. Нажми **File → New repository**
3. Заполни:
   - **Name:** `home-menu` (или любое название)
   - **Description:** `Домашнее меню 💜`
   - **Keep this code private** — ✅ **ОБЯЗАТЕЛЬНО отметь**, чтобы репозиторий был приватным!
   - **Local path:** `c:\Users\REDMI\Desktop\VScode`
4. Нажми **Create Repository**
5. GitHub Desktop предложит коммитить файлы — нажми **Commit to main**
6. Нажми **Publish repository** (синяя кнопка сверху)

---

## 4. Регистрация на Supabase

1. Открой сайт: **https://supabase.com/**
2. Нажми **Start your project** (зелёная кнопка)
3. Нажми **Sign in** → **Continue with GitHub**
4. Разреши доступ (Authorize supabase)
5. Готово! Ты в панели управления Supabase

---

## 5. Создание проекта в Supabase

1. В панели Supabase нажми **New project**
2. Выбери свою организацию (или создай — нажми **Create a new organization**)
3. Заполни:
   - **Project name:** `home-menu`
   - **Database password:** Придумай сложный пароль (запиши его!)
   - **Region:** Выбери ближайший (например, `South Korea (Seoul)` или `Singapore`)
4. Нажми **Create new project**
5. Подожди 2-3 минуты — проект создаётся

## 7. Настройка подключения

1. В панели Supabase перейди в **Settings** (⚙️ внизу слева)
2. Нажми **API** в разделе **Configuration**
3. Скопируй два значения:
   - **Project URL** (например: `https://xxxxx.supabase.co`)
   - **anon public key** (длинная строка, начинается с `eyJ...`)

4. Открой файл **`supabase-config.js`**
5. Замени эти строки на свои значения:

```javascript
const SUPABASE_URL = 'https://xxxxx.supabase.co';  // ← Вставь свой URL
const SUPABASE_ANON_KEY = 'eyJhbGc...';           // ← Вставь свой ключ
```

6. Сохрани файл (`Ctrl+S`)

## 8. Публикация через GitHub Desktop

1. Открой **GitHub Desktop**
2. Убедись что проект открыт (должен быть в списке слева)
3. Внизу слева введи:
   - **Summary:** `Подключение к Supabase`
4. Нажми **Commit to main**
5. Нажми **Push origin** (синяя кнопка сверху, или `Ctrl+P`)

---

## 9. Включение GitHub Pages

1. Открой браузер, перейди на **https://github.com/**
2. Зайди в свой репозиторий (он будет называться `твой-username/home-menu`)
3. Перейди в **Settings** (вкладка сверху)
4. В левом меню найди **Pages** (раздел Code and automation)
5. В разделе **Source** выбери **Deploy from a branch**
6. В **Branch** выбери:
   - Branch: `main`
   - Folder: `/ (root)`
7. Нажми **Save**
8. Подожди 1-2 минуты
9. Появится ссылка вида: `https://твой-username.github.io/home-menu/`
10. **Эта ссылка — твой сайт!** Нажми на неё и сайт откроется

---

## 10. Проверка работы

1. Открой ссылку сайта (из шага 9)
2. Ты должна увидеть:
   - ✅ Сайт загрузился
   - ✅ Видны 4 начальных блюда
   - ✅ Кнопки работают
   - ✅ Можно добавить новое блюдо
3. Добавь тестовое блюдо и обнови страницу (`F5`) — блюдо должно остаться
4. Если всё работает — **поздравляю, сайт в интернете!** 🎉

## 🔄 Как обновлять сайт

После каждого изменения кода:

1. Открой **GitHub Desktop**
2. Введи описание изменений в **Summary** (например: `Добавила новый раздел`)
3. Нажми **Commit to main**
4. Нажми **Push origin**
5. Сайт обновится через 1-2 минуты

## ⚠️ Важные моменты

- **Приватный репозиторий** — только ты и те, кому дашь доступ, видят код
- **GitHub Pages** — сайт доступен по ссылке всем, у кого есть ссылка
- **Supabase Free** — 500 МБ базы данных, 2 ГБ передачи (хватит на годы!)
- **Безопасность** — anon key безопасен для фронтенда, но Row Level Security защищает данные

## 🆘 Если что-то не работает

| Проблема | Решение |
|----------|---------|
| Сайт не загружается | Проверь, что в `supabase-config.js` вставлены реальные URL и ключ |
| Блюда не сохраняются | Проверь SQL-запрос в шаге 6 — таблица должна быть создана |
| Ошибка CORS | Убедись что политики RLS настроены (шаг 6) |
| 404 ошибка | Подожди 2 минуты после push — GitHub Pages обновляется не мгновенно |
