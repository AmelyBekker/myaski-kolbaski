# 📋 Инструкция для ИИ — Проект «Мясные Колбаски» (Домашнее Меню)

## 🌐 О проекте

Веб-приложение для управления домашним меню/рецептами. Пользователь может:
- Просматривать рецепты по категориям
- Выбирать блюда на «Сегодня»
- Добавлять, редактировать, удалять рецепты
- Загружать фото блюд
- Искать рецепты
- Экспортировать/импортировать данные

## 📁 Структура проекта

```
myaski-kolbaski/
├── index.html      # Основная HTML-страница
├── styles.css      # Стили
├── script.js       # Вся логика + подключение к Supabase
└── .git/           # Git-репозиторий
```

**Нет серверной части** — всё работает на клиенте через REST API Supabase.

## 🗄️ Supabase

### Подключение

- **URL:** `https://htuzloqefvljfjlapacc.supabase.co`
- **Ключ (anon):** хранится в `script.js` (строка 5)
- **Таблица:** `dishes`

### Схема таблицы `dishes`

| Колонка       | Тип     | Описание                    |
|---------------|---------|-----------------------------|
| `id`          | integer | Первичный ключ (autoincrement) |
| `category_id` | text    | Категория (first, main, side, salads, sauces, sweet, drinks, other, experiments) |
| `name`        | text    | Название блюда              |
| `recipe`      | text    | Рецепт                      |
| `image`       | text    | URL изображения             |
| `hidden`      | boolean | Скрыто/показано             |
| `in_today`    | boolean | Выбрано на сегодня          |
| `created_at`  | timestamptz | Дата создания           |

### ⚠️ ВАЖНО: snake_case ↔ camelCase

**В Supabase колонки называются:** `category_id`, `in_today`
**В JavaScript коде используется:** `categoryId`, `inToday`

При загрузке из Supabase — преобразование `snake_case → camelCase`.
При сохранении в Supabase — отправка в `snake_case` (`category_id`, `in_today`).

### RLS Policies

Для таблицы `dishes` должны быть включены RLS-политики с доступом на чтение/запись/обновление/удаление для роли `public` (anon ключ).

## 🔄 Git Workflow

### У пользователя настроен следующий процесс:

- **Репозиторий:** на **GitHub**
- **Сайт:** размещён на **GitHub Pages** (опубликован)
- **Git-клиент:** **GitHub Desktop**
- **Редактор кода:** VS Code

### После внесения изменений в код:

1. **НЕ запускай `git commit` или `git push` из командной строки** — пользователь делает это через **GitHub Desktop** вручную
2. Просто сообщи пользователю: «Закоммить и запушь изменения через GitHub Desktop»
3. GitHub Pages обновится автоматически через 1–2 минуты после пуша

### Файлы для коммита:

- `script.js` — изменения логики
- `index.html` — изменения разметки
- `styles.css` — изменения стилей

## 💾 Функции сохранения — ВАЖНО!

### `saveToSupabase()`

- Проходит по всем блюдам в массиве `dishes`
- Проверяет существование блюда по `id` (запрос `?id=eq.X`)
- Если существует → `PATCH`, если нет → `POST`
- При `POST` обновляет `id` из ответа Supabase

### `addDishToSupabase(dish)`

- Отправляет `POST` с новым блюдом
- Возвращает сохранённое блюдо с новым `id`

### `updateDishInSupabase(dish)`

- Отправляет `PATCH` по `id`

### `deleteDishFromSupabase(id)`

- Отправляет `DELETE` по `id`

## 🖼️ Изображения

- При загрузке файла — сжимаются через Canvas (300×200, quality 0.6)
- Если фото не загружено — используется заглушка с Unsplash

## 🔐 Безопасность

- `SUPABASE_ANON_KEY` хранится прямо в `script.js` — это допустимо для публичных проектов с RLS
- **НЕ используй `service_role` ключ** — он только для серверной части
- Все изменения данных проходят через RLS-политики
